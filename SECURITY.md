# Security Policy

English | [日本語](SECURITY.ja.md)

## Supported Versions

Security fixes are handled on the latest `main` branch until the project starts publishing
versioned releases.

## Reporting a Vulnerability

Please avoid posting exploit details in a public issue. If GitHub private vulnerability
reporting is enabled for this repository, use that. Otherwise, open a minimal issue asking
for a private maintainer contact and do not include sensitive details.

Useful information includes:

- macOS version
- Flickwall commit or release version
- steps to reproduce
- expected and actual behavior
- whether the issue involves file permissions, security-scoped bookmarks, or wallpaper
  application
