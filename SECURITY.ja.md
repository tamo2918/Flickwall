# セキュリティポリシー

[English](SECURITY.md) | 日本語

## サポート対象バージョン

正式なバージョン付きリリースを開始するまでは、最新の `main` ブランチで
セキュリティ修正を扱います。

## 脆弱性の報告

悪用可能な詳細を公開 Issue に投稿しないでください。このリポジトリで GitHub の
private vulnerability reporting が有効になっている場合は、それを使用してください。
有効でない場合は、機密情報を含めずに、メンテナーへ非公開で連絡する方法を尋ねる
最小限の Issue を作成してください。

報告に含めると役立つ情報は次のとおりです。

- macOS バージョン
- Flickwall のコミットまたはリリースバージョン
- 再現手順
- 期待した挙動と実際の挙動
- ファイル権限、セキュリティスコープ付きブックマーク、壁紙適用処理のどれに関係するか
